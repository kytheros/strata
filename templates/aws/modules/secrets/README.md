# `aws/modules/secrets` — Secrets Manager wrapper with rotation + per-secret CMK

**Tool choice: Terraform (OpenTofu compatible).** Reason: matches the rest of the AWS deploy template; the rotation/KMS/IAM wiring is heavy on resource-graph edges that Terraform's plan model surfaces clearly. CDK would add a runtime dependency for a thin wrapper that's mostly declarative.

## What this creates

| Resource | When | Notes |
|---|---|---|
| `aws_secretsmanager_secret` | always | Path-style name `strata/{env}/{secret_name}`. KMS-encrypted. Recovery window ≥ 7 days (AWS minimum); module never exposes `force_delete_without_recovery`. |
| `aws_kms_key` (CMK) | when `var.kms_key_id == ""` | Per-secret CMK with rotation enabled, 7-day deletion window. Key policy allows account-root admin (scoped to this key only) + Secrets Manager service principal + optional rotation Lambda principal. |
| `aws_kms_alias` | when CMK is created | `alias/strata-secret-{env}-{name}`. Slashes in `secret_name` are normalized to hyphens because alias names disallow `/`. |
| `aws_secretsmanager_secret_version` | when `var.create_initial_version = true` | Seeds AWSCURRENT with `var.initial_value` (sensitive). `lifecycle.ignore_changes` prevents Terraform from fighting a rotation Lambda that overwrites the value. |
| `aws_secretsmanager_secret_rotation` | when `var.rotation_lambda_arn != ""` | Wires the existing Lambda as the rotator. `automatically_after_days = var.rotation_days`. |
| `aws_lambda_permission` (rotation invoke) | when rotation enabled | Grants `secretsmanager.amazonaws.com` permission to invoke the rotation Lambda for this specific secret ARN. |
| `aws_secretsmanager_secret_policy` | when rotation enabled | Resource-based policy granting the rotation Lambda's exec role `Describe/Get/Put/UpdateSecretVersionStage` on this secret only. |

All resources tagged `Project=strata`, `Component=secrets`, `ManagedBy=terraform`, `Environment={env_name}`, `SecretName={secret_name}`.

## Why per-secret CMK by default

Three options for encryption:

1. **AWS-owned key (`aws/secretsmanager`)** — free, but key policy is opaque; can't audit who can decrypt.
2. **Per-service CMK** — one key shared across N secrets in a service; auditable but blast radius wider.
3. **Per-secret CMK** ← **module default** — tightest blast radius; key policy is auditable per secret; ~$1/CMK/mo.

The default is (3) because Strata holds a small number of secrets per env (~5–10), so the cost is negligible (~$5–10/env/mo for keys, dominated by NAT and endpoints). For high-fanout cases (e.g., one secret per tenant) pass an existing CMK ARN via `var.kms_key_id` to fall back to (2) or (1).

## When to set `create_initial_version`

| Scenario | `create_initial_version` |
|---|---|
| Static API key (Stripe, Polar, OpenAI) bootstrap-seeded then never rotated | `true` |
| Aurora master credential — auto-generated by rotation Lambda on first run | `false` |
| Internal HMAC signing key, manually seeded once, replaced via `aws secretsmanager update-secret` | `true` |
| Federated OIDC client secret managed elsewhere; this module just stores it | `true` |

When `false`, the secret resource is created without a value; consumers must wait for the rotation Lambda's first run (or write the value via the AWS API). The Aurora module (`AWS-1.4`) uses this pattern.

## Inputs

See `variables.tf` for the full list.

| Variable | Required? | Default | Notes |
|---|---|---|---|
| `env_name` | yes | — | `dev|staging|prod`. |
| `aws_region` | no | `us-east-1` | Used in the `kms:ViaService` condition. |
| `secret_name` | yes | — | Lowercase letters/digits/hyphens/slashes only. Becomes part of `strata/{env}/{name}`. |
| `description` | yes | — | Required (no default). Surfaces in the Secrets Manager console. |
| `kms_key_id` | no | `""` | Empty → module creates a per-secret CMK. Otherwise pass an existing key ARN/ID/alias. |
| `recovery_window_days` | no | `7` | Range `[7, 30]`. Module does not expose immediate-delete. |
| `create_initial_version` | no | `false` | When true, seeds AWSCURRENT with `var.initial_value`. |
| `initial_value` | no | `""` | Sensitive. Only consumed when `create_initial_version = true`. |
| `rotation_lambda_arn` | no | `""` | Empty → no rotation. Otherwise the module attaches the Lambda + grants it access. |
| `rotation_days` | no | `30` | Only consumed when rotation is enabled. |
| `extra_tags` | no | `{}` | Merged into the default tag set. |

## Outputs

`secret_arn`, `secret_name`, `secret_id`, `secret_version_id` (empty when no initial version), `kms_key_arn`, `kms_key_alias` (empty when an external key was supplied), `rotation_enabled`, `consumer_iam_policy_json`.

The `consumer_iam_policy_json` output is the load-bearing one: attach it to a consumer task role to grant least-privilege read on this secret + decrypt on its CMK. Example consumer pattern:

```hcl
module "db_password" {
  source      = "../../modules/secrets"
  env_name    = "dev"
  secret_name = "aurora-master"
  description = "Aurora master credential, rotated 30d via the AWS-managed rotation Lambda."
  rotation_lambda_arn = aws_lambda_function.aurora_rotator.arn
}

resource "aws_iam_role_policy" "task_read_db_secret" {
  name   = "read-db-secret"
  role   = aws_iam_role.task.id
  policy = module.db_password.consumer_iam_policy_json
}
```

## Security notes

- **Never use `force_delete_without_recovery`.** Not exposed by this module. A 7-day window is the minimum AWS allows; recovering a "deleted" secret is a one-line API call within the window. Recovering one that was force-deleted is a credential rebuild.
- **`secret_string` is sensitive end-to-end.** `var.initial_value` is `sensitive = true`, so it doesn't appear in plan output. The version resource's `secret_string` field is never exported by an output. The state file still contains it (Terraform's state model has no way around this) — so the state backend's S3 bucket must be SSE-encrypted with versioning + access logging on, which the bootstrap module guarantees.
- **Account-root in the KMS key policy is intentional.** AWS-recommended escape hatch — without it, a key can become unmanageable if all IAM admins are revoked. The statement is scoped to this key only (a KMS key policy's `Resource: "*"` always means "this key", not all keys). The `checkov:skip` annotations in `main.tf` document this.
- **Rotation Lambda principal is least-privilege.** The Lambda only gets KMS Decrypt + GenerateDataKey scoped via `kms:ViaService = secretsmanager.<region>.amazonaws.com`, and only the four `secretsmanager:*` actions required by the rotation contract.
- **No secret material in local-exec or templates.** The module never reads `var.initial_value` into anything other than `aws_secretsmanager_secret_version.secret_string`.

## Cost

| Component | Monthly |
|---|---|
| Secret (one) | ~$0.40 + $0.05 per 10k API calls |
| Per-secret CMK (when created by module) | ~$1.00 + $0.03 per 10k calls |
| **Per secret with default per-secret CMK** | **~$1.40 / month at idle** |

For 10 secrets in dev: ~$14/month. Negligible compared to NAT (~$66) and VPC endpoints (~$72). For very-high-fanout cases (>30 secrets per env) consider passing a shared `var.kms_key_id` to drop the CMK cost.

## Reviewers required before apply

- **`security-compliance`** — KMS key policy (account-root admin + service principal + rotation Lambda); IAM grant to the rotation Lambda's role; recovery-window choice.
- **`finops-analyst`** — Per-secret CMK default vs shared-key alternative for the consumer's specific secret count.

## Verification (post-apply)

```bash
# Secret exists with the expected name
aws secretsmanager describe-secret --secret-id strata/dev/<your-secret> \
  --query '{Name:Name,KmsKeyId:KmsKeyId,RotationEnabled:RotationEnabled,RotationRules:RotationRules}'

# CMK exists with rotation enabled (when module-created)
aws kms describe-key --key-id alias/strata-secret-dev-<your-secret> \
  --query 'KeyMetadata.{KeyState:KeyState,KeyManager:KeyManager,KeyRotationEnabled:KeyRotationEnabled}'

# Confirm consumer policy renders to expected JSON
terraform output -raw consumer_iam_policy_json | jq .

# Trigger a rotation (with-rotation example only)
aws secretsmanager rotate-secret --secret-id strata/dev/<your-secret>
aws logs tail /aws/lambda/<rotation-lambda-name> --since 5m
```
